# Content Extraction

三池提取规则：如何从不同格式的日记中提取 Ideas、Seeds 和 Insights。

## 支持格式

**完全支持自由格式日记**。

无论日记是否使用模板，都可以进行三池提取。系统通过语义分析识别内容类型，而非依赖固定标题。

## 三池定义

| 池子 | 定义 | 典型内容 |
|------|------|----------|
| **Ideas Backlog** | 可执行的想法、创意、待办灵感 | 产品想法、改进方案、学习主题 |
| **Content Seeds** | 内容创作的原始素材 | 金句、故事片段、观察、引用 |
| **Insights Pool** | 洞察、反思、认知升级 | 顿悟、模式识别、经验教训 |

## 提取规则

### Ideas Backlog

**识别信号**:
- 包含"想尝试"、"可以"、"应该"、"计划"等词
- 疑问句形式的探索性问题
- 以动词开头的行动项
- 明确提到"想法"、"Idea"

**示例输入**:
```
今天想到一个可以做的小工具：自动整理Obsidian笔记的脚本
```

**提取结果**:
```markdown
- [ ] #idea/tool id:idea-20260226-a3f7b2
  - **想法：** 自动整理Obsidian笔记的脚本
  - **来源：** [[2026-02-26]]
  - **添加日期：** 2026-02-28
```

### Content Seeds

**识别信号**:
- 引用的他人话语
- 具体场景/故事的描述
- 观察到的现象
- 数据点或统计
- 有趣的事实

**示例输入**:
```
今天在咖啡馆听到两个人讨论远程工作的痛点，其中一个说：
"最大的问题是边界感消失，家变成了办公室"
```

**提取结果**:
```markdown
### Seed #202602260001 id:seed-20260226-b8c2d1
- **素材：** "最大的问题是边界感消失，家变成了办公室" - 远程工作者的痛点
- **来源：** [[2026-02-26]]
- **添加日期：** 2026-02-28
```

### Insights Pool

**识别信号**:
- 包含"意识到"、"发现"、"明白"、"原来"等词
- 对比过去和现在的认知变化
- "应该"到"原来是"的转变
- 对模式/规律的总结

**示例输入**:
```
突然意识到，我之所以拖延不是因为懒，而是因为恐惧失败。
每次开始新任务时那种"万一做不好怎么办"的焦虑才是真正的阻力。
```

**提取结果**:
```markdown
### 2026-02-28 id:insight-20260226-f2a9e3
**Insight:** 拖延的根源不是懒惰，而是对失败的恐惧和对任务质量的焦虑
- **来源：** [[2026-02-26]]
- **添加日期：** 2026-02-28
```

## 自由格式日记处理

### 无模板日记示例

**原始日记**:
```markdown
# 2026-02-26

今天过得挺充实的。早上读完了《深度工作》，里面有个观点很戳我：
"忙碌是一种懒惰——懒得思考什么是真正重要的"

想到一个点子：可以做一个浏览器插件，统计每天在各网站上的时间，
然后生成一份"注意力报告"。技术栈可以用Chrome Extension + SQLite。

另外意识到最近睡眠质量差可能是因为睡前刷手机。大脑把床和清醒
联系在一起了。也许应该把充电放在客厅，强制睡前不碰手机。

下午开了3个会，效率一般。明天要试试把会议集中放在下午，
上午留大块时间做深度工作。
```

**提取结果**:

**Ideas Backlog**:
```markdown
- [ ] #idea/tool id:idea-20260226-x7k2m1
  - **想法：** 浏览器插件 - 统计每日网站时间并生成"注意力报告"，技术栈：Chrome Extension + SQLite
  - **来源：** [[2026-02-26]]
  - **添加日期：** 2026-02-28

- [ ] #idea/workflow id:idea-20260226-p9n4q8
  - **想法：** 把会议集中放在下午，上午留大块时间做深度工作
  - **来源：** [[2026-02-26]]
  - **添加日期：** 2026-02-28
```

**Content Seeds**:
```markdown

- **素材：** "忙碌是一种懒惰——懒得思考什么是真正重要的" - 《深度工作》

- **添加日期：** 2026-02-28
```

**Insights Pool**:
```markdown
### 2026-02-28 id:insight-20260226-w8e2c4
**Insight:** 睡眠质量差可能是因为睡前刷手机导致大脑把床和清醒联系在一起
- **来源：** [[2026-02-26]]
- **添加日期：** 2026-02-28
```

## 模板日记 vs 自由格式

### 模板日记（推荐）

如果有固定模板，提取准确率更高：

```markdown
## 今天的状态
8/10

## 一个想法
想做一个小工具...

## 一个观察
发现很多人...

## 一个洞察
意识到其实...
```

### 自由格式日记

无固定结构，依赖语义分析：

```markdown
今天想到...
另外发现...
还有...
```

**两者都支持**，模板日记只是让提取更精准。

## 不提取的内容

以下类型内容**不会**进入三池：

- 纯情绪宣泄（无洞察）
- 流水账式记录（无想法/素材/洞察）
- 已明确标记为"完成"的事项
- 与具体日期强绑定的临时信息

## 人工复核建议

虽然系统会自动提取，但建议定期人工复核 Pool 文件：

1. **删除**低质量或不再相关的条目
2. **合并**重复或相似的条目
3. **调整**分类（如将某个 Idea 移动到 Seeds）
4. **补充**上下文或链接

Pool 文件是 Living Document，不是只进不出的垃圾桶。
